# Roadmap: 藍牙音訊裝置功能

## 概覽

為現有 Mobile OS Versions Flutter App 新增藍牙音訊裝置頁面，透過兩階段交付完整功能：首先建立核心顯示功能（裝置資訊、當前 codec 參數、權限處理），接著新增支援清單與引導式切換體驗，讓用戶能查看並透過系統開發者選項調整藍牙音訊編碼參數。

## Phases

**階段編號規則：**
- 整數階段 (1, 2, 3)：已規劃的里程碑工作
- 小數階段 (2.1, 2.2)：緊急插入的階段（標記為 INSERTED）

小數階段依數值順序出現在其周圍整數之間。

- [ ] **Phase 1: 核心顯示功能** - 建立導航、權限處理、裝置與 codec 資訊顯示
- [ ] **Phase 2: 支援清單與引導切換** - 顯示支援的 codec 清單並引導用戶切換參數

## Phase Details

### Phase 1: 核心顯示功能
**目標**: 用戶可以查看已連接的藍牙音訊裝置詳細資訊與當前 codec 參數
**依賴**: 無（第一階段）
**需求**: NAV-01, PERM-01, PERM-02, PERM-03, DEVICE-01, DEVICE-02, DEVICE-03, DEVICE-04, CODEC-01, CODEC-02, CODEC-03, CODEC-04, CODEC-05
**成功標準** (必須為真的狀態):
  1. 用戶可以從左側選單進入藍牙音訊頁面
  2. 用戶可以看到已連接藍牙裝置的名稱、MAC 位址、藍牙版本
  3. 用戶可以看到當前使用的 codec 類型（SBC/AAC/LDAC/APTX 等）
  4. 用戶可以看到當前 codec 的取樣率、位元深度、bitrate、聲道模式
  5. 無藍牙裝置連接時，用戶看到清楚的空狀態提示訊息
  6. Android 12+ 用戶被正確引導完成 BLUETOOTH_CONNECT 權限請求
  7. Android 8.0 以下用戶看到不支援的提示訊息
**計劃數**: 待定

Plans:
- [ ] 01-01: 待規劃
- [ ] 01-02: 待規劃
- [ ] 01-03: 待規劃

### Phase 2: 支援清單與引導切換
**目標**: 用戶可以查看裝置支援的 codec 清單，並透過引導流程切換 codec 參數
**依賴**: Phase 1
**需求**: SUPPORT-01, SUPPORT-02, SWITCH-01, SWITCH-02, SWITCH-03, SWITCH-04
**成功標準** (必須為真的狀態):
  1. 用戶可以看到手機支援的所有 codec 清單
  2. 用戶可以看到當前連接耳機支援的所有 codec 清單
  3. 用戶點擊切換 codec 按鈕時，被正確引導至系統開發者選項
  4. 用戶可以在應用內看到如何啟用開發者選項的說明
  5. 用戶可以在應用內看到如何切換取樣率、位元深度、bitrate 的說明
**計劃數**: 待定

Plans:
- [ ] 02-01: 待規劃
- [ ] 02-02: 待規劃
